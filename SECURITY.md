 # Politique de sécurité (Security Policy)                                                                                
                                                                                                                             
    Bienvenue sur le projet **Jeu d'Échecs IA**. La sécurité de notre application et des données de nos utilisateurs est une 
  priorité. Cette page décrit les versions prises en charge et la marche à suivre pour signaler une vulnérabilité.           
                                                                                                                             
    ## Versions prises en charge (Supported Versions)                                                                        
                                                                                                                             
    S'agissant d'une application s'exécutant exclusivement côté client (front-end) dans le navigateur, seule la version la   
  plus récente disponible sur la branche principale (`main`) bénéficie activement des correctifs de sécurité.                
                                                                                                                             
    | Version | Prise en charge |                                                                                            
    | ------- | --------------- |                                                                                            
    | main    | ✅ Oui (Actif)   |                                                                                           
    | < 2.0.0 | ❌ Non          |                                                                                            
                                                                                                                             
    ---                                                                                                                      
                                                                                                                             
    ## Signaler une vulnérabilité (Reporting a Vulnerability)                                                                
                                                                                                                             
    Si vous découvrez une faille de sécurité ou une vulnérabilité dans cette application, **merci de ne pas ouvrir de ticket 
  d'incident (Issue) public**. Les signalements publics peuvent exposer inutilement l'application avant qu'un correctif ne   
  soit déployé.                                                                                                              
                                                                                                                             
    Veuillez suivre la procédure suivante :                                                                                  
                                                                                                                             
    1. Envoyez un e-mail privé à l'adresse suivante : **[Insérez votre e-mail ici]** (ou contactez le mainteneur via GitHub).
    2. Décrivez précisément la vulnérabilité ainsi que les étapes nécessaires pour la reproduire (si possible en fournissant 
  un script de test ou une capture d'écran).                                                                                 
                                                                                                                             
    ### Processus de traitement :                                                                                            
    * **Accusé de réception** : Nous nous efforçons de répondre à votre e-mail sous **48 à 72 heures** ouvrées pour confirmer
  la réception et l'analyse de votre signalement.                                                                            
    * **Résolution** : Une fois la faille validée, nous travaillerons sur un correctif qui sera déployé directement sur la   
  branche `main`.                                                                                                            
    * **Divulgation** : Une fois le correctif en ligne, nous publierons une annonce ou un avis de sécurité pour documenter la
  résolution de la faille en vous attribuant le crédit du signalement si vous le souhaitez.                                  
                                                                                                                             
    Merci pour votre aide précieuse dans l'amélioration de la sécurité de ce projet !
